WEB移动端开发

发表于 2017-05-01 | | 阅读次数
| 字数统计 4,823 | 阅读时长 21

详细讲解了移动端开发的各种问题以及方法，通过学习本文章就学会了移动端开发。
个人笔记，仅供参考。

常用meta设置

viewport 视口(可视区窗口)

<meta name="viewport" content="width=device-width,user-scalable=no,initial-scale=.5,minimum-scale=.5,maximum-scale=1">

默认不设置viewport一般可视区宽度在移动端是980
width 可视区的宽度 (number||device-width) 设备本身宽度
user-scalable 是否允许用户缩放 (yes||no) iOS10无效 (我们放在事件章节解决)
initial-scale 初始缩放比例
minimum-scale 最小缩放比例
maximum-scale 最大缩放比例

devicePixelRatio 像素比

n = window.devicePixelRatio

像素比把一个像素 放大至 N个像素去显示

X5内核可以禁止横屏浏览，比如QQ，UC

QQ强制横屏或者竖屏显示

<meta name="x5-orientation" content="portrait|landscape" />

QQ设置全屏

设置之后会隐藏掉头部信息和地址栏，底部的菜单栏。类似于PC端F11

<meta name="x5-fullscreen" content="true" />

UC强制横屏或者竖屏显示

<meta name="screen-orientation" content="portrait">

UC全屏显示

<meta name="full-screen" content="yes">

禁止识别电话号码和邮箱号码

<meta name="format-detection" content="telephone=no, email=no" />

可以手动添加识别

<p>13888888888</p>                  //禁止识别之后，这堆数字就无法点击弹出通话按钮

<a href="tel:18888888888">请拨打电话18888888888</a>      //可以通过手动来设置实现识别功能
<a href="mailto:motao@motao.com">请发送邮件</a>

默认样式等问题

在移动端中，有一些默认样式不够美观，我们希望清除或者修改它

清楚点击阴影

比如a,input,button这些标签点击会有阴影出现,可以通过-webkit-tap-highlight-color修改

a,input,button { -webkit-tap-highlight-color:rgba(0,0,0,0); }

清楚按钮圆角

在移动端，用border-raduis: 0;清除圆角是没效果的，一定要用webkit-appearance: none;来清除。
webkit-appearance: none;清除之后还是会有一点圆角，这个时候再结合border-radius:0;来实现完全清除。

input,button { -webkit-appearance: none;  border-radius: 0;}

默认字体设置

在PC端我们一般会设置宋体或者微软雅黑这两种系统自带的默认字体。
移动端，默认字体只有一个英文字体Helvetica。

body { font-family: Helvetica; }

选中文字设置

避免移动端长按选中文字

body { -webkit-user-select: none; }

但是这段代码的兼容性不好，目前测试在IOS下有效果，但是安卓下面会失效。还有比如长按文字弹出底部菜单，这些的解决办法我们会在事件章节中详细解说。

选中文字缩放

为了避免用户修改我们的文字大小，需要用-webkit-text-size-adjust禁止掉。

body * { -webkit-text-size-adjust: 100%; }

Font Boosting问题

在一段文字我们没有给它设置高度的时候，在webkit内核下，文字的大小被浏览器放大了。这种问题不是每次都出现，但是有时候还是会出现，这就是Font Boosting问题。

解决办法：

  1. 设置高度，如果是需要用户来编辑，那么我们可以先设置一个最大高度。
  2. 设置最大高度 max-height

Fixed定位的问题

移动端在以前的系统版本里是不支持固定定位的，目前来说已经开始慢慢支持了，但是支持的不是很好，具体解决办法需要放到事件章节里解决。这里我们列出一个不用JS解决的办法，但是支持也不是很完美，权作参考。

Viewport适配

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>Document</title>
<script type="text/javascript">
(function(){
    var w = window.screen.width;  //获取当前屏幕的尺寸
    var targetW = 320;              //目标尺寸
    var scale = w/targetW; //当前尺寸/目标尺寸
    var meta = document.createElement("meta");
    meta.name = "viewport";
    meta.content = "user-scalable=no,initial-scale="+scale+",minimum-scale="+scale+",maximum-scale="+scale+""
    document.head.appendChild(meta);
})();
</script>

弹性盒模型

使用弹性盒模型的时候，父元素必须加 display:box或 display:inline-box

box-orient 定义盒模型的主轴方向

box-direction 元素排列循序

box-pack 横轴方向富裕的空间管理

新版设置 justify-content: space-around;,可以让富裕空间平均分配在每个元素两侧。
老版没有或者功能，相对于新版来说，缺失了这个功能。

box-pack 竖轴方向富裕的空间管理

  • box-flex 定义盒子的弹性空间
    子元素的尺寸 = 盒子的尺寸 * 子元素的box-flex属性值 / 所有子元素的box-flex属性值的和

box-ordinal-group 设置元素的具体位置

移动端事件

touchstart：手指触摸
touchmove：手指移动
touchend：手指离开

在移动端开发的时候，浏览器的模拟器时好时坏，一般不用on的方式绑定事件函数，要用事件绑定的方式(addEventListener)。
pc上的事件比移动端的事件略慢，大概是在300ms左右。

移动端的点透

当上层元素发生点击的时候，下层元素也有点击（焦点）特性，
在300ms之后，如果上层元素消失或者隐藏，目标点就会“漂移”到
下层元素身上，就会触发点击行为。

解决：

  1. 下层不要使用有点击（焦点）特性的元素。
    但是如果给下层的元素添加了点击事件还是会有点透行为

p元素上的touchstart事件依然会被触发。

  1. 阻止pc事件。
    阻止document的touchstart默认事件之后，a标签的点击事件也会被取消。
    所以需要手动给a标签添加点击跳转的事件。

事件对象

当给某个元素加上了事件绑定函数之后，事件函数默认的第一个参数就是事件对象
事件对象：

  • 当用户在浏览器下触发了某个行为，事件对象会记录用户操作时一些细节信息。
  • touches 当前位于 屏幕上的所有手指的一个列表
  • targetTouches 位于当前DOM元素上的手指的一个列表
  • changedTouches 涉及当前事件的手指的一个列表

在开发过程中，发现一个问题，获取手指的位置的时候不能用event.pageX, pageX是基于文档的，在这里发现失效了。改用clientX之后才能获取。

transform详解

这个章节主要是理解矩阵的内容。

经过测试，在移动端开发中，transform最好还是加上前缀 -webkit-transform，大部分浏览器都还是支持加前缀的方式。

transform的执行顺序是先写的后执行。

在移动端是不推荐使用top、left等等去做运动，推荐使用transform去完成运动效果，transform相对于top、left等值性能会好很多。

关于矩阵

获取transform的属性值
如果我们用 getComputedStyle()去获取transform的属性，只会获取到一个矩阵。
例如： matrix(0.5, 0, 0, 0.5, 50, 0);
transform下的属性是不能通过这种这方式获取的，我们可以采用obj.style.transform来获取。

然后通过正则的方法来获取其中的数值,不过这种方法非常麻烦。

transform的matrix的默认值是matrix(1,0,0,1,0,0),也就是说把matirx(1,0,0,1,0,0)赋给transform是不会有任何变化的。

矩阵matrix参数

使用matrix矩阵的方法来实现上面的transform 2D效果。

matrix( 1,0,0,1,0,0 )
matrix( a,b,c,d,e,f )
位移
  • X轴位移 = e + x;
    matrix(1,0,0,1,100,0)相当于X轴移动了100px；
  • Y轴位移 = f + x;
    matrix(1,0,0,1,0,100)相当于Y轴移动了100px；
缩放

  • X轴缩放：

    • a = a * x; a缩放的倍数
    • c = c * x; c缩放的倍数
    • e = e * x; e缩放的倍数
    • matrix(.5,0,0,1,50,0)`相当于X轴先缩放0.5倍然后位移了100px；
      a缩放0.5倍。c是零，零乘以0.5还是零。e是X轴位移了100，但是也要缩放0.5，所以是50；

  • Y轴缩放

    • b = b * x; a缩放的倍数
    • d = d * x; c缩放的倍数
    • f = f * x; e缩放的倍数
    • matrix(.5,0,0,.5,0,0)`相当于XY轴同时缩放了0.5倍；
先位移，再缩放
  • 位移- matrix(1,0,0,1,100,100); XY轴同时位移100px;
  • 缩放- matrix(.5,0,0,.5,50,50); XY轴同时缩放0.5倍;
先缩放，再位移
  • 缩放- matrix(.5,0,0,.5,0,0); XY轴同时缩放0.5倍
  • 位移- matrix(.5,0,0,.5,100,100); XY轴同时位移100px;

请记住一点，transform的执行顺序是先写的后执行。

这就能解释出为什么transform书写顺序不同，执行的结果不同了。

斜切

角度转弧度公式 = deg*Math.PI/180
正切 = Math.tan

  • X斜切：(deg)

    • c = Math.tan(deg Math.PI/180);
      斜切是使用一个三角函数来计算，括号里面不能直接写角度，需写一个弧度，所以在这之前，我们需要把角度转成弧度
      上面已经写了角度转弧度公式-角度       Math.PI / 180

  • X斜切：(deg)

    • b = Math.tan(deg*Math.PI/180);
旋转

旋转需要修改4个参数。使用正选、余弦以及角度转弧度公式来计算。跟斜切有一定的差别
角度转弧度公式 = deg*Math.PI/180
正弦 = Math.sin
余弦 = Math.cos

  • a = Math.cos(deg/180*Math.PI);
  • b = Math.sin(deg/180*Math.PI);
  • c = Math.sin(deg/180*Math.PI);
  • d = Math.cos(deg/180*Math.PI);

3D

  • perspective – 景深 （我在多远之外来看这个元素）
    景深要加给要做3d变化的元素的父级。 perspective: 200px;

  • perspective-origin: center center; – 视线方向

  • rotateX / rotateY / rotateZ / rotate3D – 旋转
    rotateZ 视觉上相当于2D的 rotate

  • translateX / translateY / translateZ / translate3d – 位移
    translataZ 视觉上相当于 scaleX与 scaleY 同时缩放。

  • transform-style – 元素在做3D变换的时候，是否保留了子元素的3D变换。

    • 不保留：在子元素上设置 transform-style: flat;(直接不设置，默认是不保留的)
    • 保留 ：在子元素上设置 transform-style: preserve-3d;

  • backface-visibility – 隐藏背面 backface-visibility: hidden;
