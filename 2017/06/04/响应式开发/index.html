有用的文件

404.html
在每个项目中，我们最好新建一个404.html文件，以方便服务器出现404错误的时候，跳转到这个页面。这个页面可以自己定制，会比浏览器默认的404页面更个性化，也更方便设计。
robots.txt
新建一个robots.txt文件，当搜索爬虫访问到一个网站的时候，首先会检查该网站是否有一个这样的文件，也就是robots.txt文件。如果存在，搜索引擎就会根据该文件的定制来确定搜索的范围。如果不存在，搜索引擎就会抓取该网站的所有内容，包括我们不希望被用户看到的口令内容。
一般网站都会有一些后台管理页面不希望被搜索引擎抓取，这个时候我就可以通过robots.txt文件告诉搜索引擎不要抓取。

更多的设置内容，可以通过百度谷歌来搜索。而且每个搜索引擎都会有不同的设置方式，还有一些流氓的搜索引擎会无视这个设定。因为这个文件只是相关行业的一个约定，如果不遵守别人也拿你没办法。

humans.txt
新建一个humans.txt文本文件，通过这个文件，我们可以快速的了解到这个网站的团队信息，开发人员信息，开发中的故事等等。 robots.txt文件是给机器爬虫看的，而humans.txt是给人看的。
这个文件理论是想怎么写就怎么写，但是如果想要有个参照可以去 humanstxt官网查看。

.editorconfig
.editorconfig官网
这个文件是用于统一代码规范的文件，很多项目里有有用到。这个文件可以帮助开发人员在不同的编码器以及IDE保持统一的代码风格。

.gitignore
这个文件是用github版本管理工具的。可以设置要忽略掉的文件，以保证每次提交版本库的时候不用提交一些无用的文件。如日志，或者一些工具生成的发布文件什么的。

README.md
文件下可以写入 项目简介、使用方式、相关链接

CHANGLOG.md
项目的每个版本更新
说明版本号，更新内容，修复了那些问题等

视口和屏幕的区别

在PC端，只有一个视口，这个视口是显示主窗口的区域。
但是在移动端却有3个视口：

  • 布局视口(layout viewport)
    网页实际的宽度
  • 可视视口(visual viewport)
    根据用户放大缩小经行改变，不固定值，默认是手机屏幕的宽度
  • 理想视口(ideal viewport)
    理想视口就是布局视口在一个设备上的最佳尺寸，为构建手机浏览器优化的页面而添加的。
    理想视口下的页面便于浏览器浏览、阅读。不需要缩放就能很好的查看浏览页面。

实现理想视口需要在页面添加视口元标签。

响应式要点

单位

em

相对的长度单位

  1. em相对参照物为父元素的font-size
  2. em具有继承的特点
  3. 当没有设置font-size时，浏览器会有一个默认设置的em设置：1em = 16px
  4. em的缺点：容易混乱

rem

  1. rem的相对参照物为根元素html，相对于参照固定不变，所以比较好计算
  2. 当没有设置font-size时，浏览器会有一个默认的rem
    设置1rem = 16px，这点和em是一致的

使用了媒体查询之后，如果用户手动修改字体之后，媒体查询的值会有变化。
比如设定为 @media screen and (max-width : 800px){},800px以内的代码回在文字放大之后，也许到1200PX以内就会有变化。
我们可以使用em的方式来设定媒体查询，让断点随着文字的大小变化而变化。
由于媒体查询的优先级非常高，它读取的默认字体大小是浏览器的16px，所以我们不能按html 设定的 font-size:62.5%;来将媒体查询单位转成rem。
我们应该按照浏览器默认字体16px，用em来转换媒体查询。比如: @media screen and (max-width : 50em){}; 800/16 = 50em

注意： 如果网站针对特定的分辨率有功能的不同，那么建议断点都设置成PX的绝对数值。这样可以避免用户修改字体大小导致的网站功能不可用的情况。 如果网站只是展示样式上的不同，但是功能都一样的话，推荐使用媒体查询相对的长度单位，可以会让用户的体验非常好。

如果只设置srcset的话，会有一个小小的坑。假如图片的宽度只有屏幕宽度的50%，当屏幕大于480的时候，浏览器就会默认渲染800的图片，而实际上我们只需要480的图片。
这是因为我们没有设置sizes属性，虽然我们没有设置，但是浏览器还是默认设置了 sizes = 100vw。这里的vw代表了设备屏幕的宽度view width,100vw代表了100%的屏幕宽度。

所以要避免这个坑，我们还需要设置sizes属性，sizes属性和srcset一样都是用逗号隔开。第一个代表了媒体查询，第二个值代表了图片预估的尺寸。也就是说在这中媒体媒体条件下，它的图片应该是什么尺寸的。注意这两个值是用空格隔开的


sizes的媒体查询是可以省略掉的，这里代码表示，在任何情况下图片都是以50vw的尺寸显示，也就是以屏幕的50%宽度显示，也就是需要在屏幕为960的时候才会显示800的图片。
