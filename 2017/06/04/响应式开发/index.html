主要是为了防止老旧的浏览器，不支持带媒体属性的查询，而应用到给定的样式

所以旧浏览器一定要加上only。
加only，老的浏览器不认识，后面的代码不会执行，媒体查询不起作用。
不加only，老的浏览器会认识screen，但是不认识后面的条件，会忽略条件，把媒体铲鲟里面的东西全部应用，可能会导致页面混乱。

视口和屏幕的区别

在PC端，只有一个视口，这个视口是显示主窗口的区域。
但是在移动端却有3个视口：

  • 布局视口(layout viewport)
    网页实际的宽度
  • 可视视口(visual viewport)
    根据用户放大缩小经行改变，不固定值，默认是手机屏幕的宽度
  • 理想视口(ideal viewport)
    理想视口就是布局视口在一个设备上的最佳尺寸，为构建手机浏览器优化的页面而添加的。
    理想视口下的页面便于浏览器浏览、阅读。不需要缩放就能很好的查看浏览页面。

实现理想视口需要在页面添加视口元标签。

响应式要点

单位

em

相对的长度单位

  1. em相对参照物为父元素的font-size
  2. em具有继承的特点
  3. 当没有设置font-size时，浏览器会有一个默认设置的em设置：1em = 16px
  4. em的缺点：容易混乱

rem

  1. rem的相对参照物为根元素html，相对于参照固定不变，所以比较好计算
  2. 当没有设置font-size时，浏览器会有一个默认的rem
    设置1rem = 16px，这点和em是一致的

工具样式

bootstrap响应式

首先在头部要引入bootstart.css

  1. 布局容器
    • container 固定宽度
    • container-fluid （100%宽度）
  2. row
    • “行（row）”必须包含在.container（固定宽度）或 .container-fluid（100%宽度）中
  3. column
    • 只有”列（column）”可以作为行（row）的直接子元素
  4. 栅格参数
    • .col-xs- : 超小屏幕手机 (<768px)
    • .col-sm- : 小屏幕 平板 (>=768px)
    • .col-md- : 中等屏幕 左面显示器(>=992px)
    • .col-lg- : 大屏幕 大桌面显示器(>=1200px)
  1. 列偏移
    • .col-md-offset-* 类可以将列向右侧偏移

  1. 列排序

    • .col-md-push- 和 .col-md-pull-

  2. 显示隐藏

    • visible-md 、hidden-md
